Configuração de Cobrança

Use o Mercado Pago como gateway principal para oferecer Pix, cartão de crédito e boleto no Brasil. O Stripe pode ficar como opção complementar para checkout hospedado e assinaturas.

Importante: as chaves secretas devem ficar nas variáveis de ambiente do servidor, nunca no navegador. O bloqueio automático da loja acontece quando a licença vencer e a cobrança continuar em atraso.

1. Mercado Pago

  1. Crie sua conta de vendedor no Mercado Pago.
  2. No painel de desenvolvedores, gere Access Token e Public Key.
  3. No servidor, configure: MERCADOPAGO_ACCESS_TOKEN e MERCADOPAGO_PUBLIC_KEY.
  4. Ative Checkout Transparente/Checkout API para usar Pix, boleto e cartão no checkout.
  5. Configure o webhook do Mercado Pago apontando para o seu servidor.

2. Stripe

  1. Use Stripe se quiser complementar com checkout hospedado e outras funções.
  2. Configure: STRIPE_SECRET_KEY, STRIPE_PUBLISHABLE_KEY e STRIPE_WEBHOOK_SECRET.
  3. Para Brasil, valide no seu painel da Stripe a disponibilidade dos meios locais que pretende ativar.

3. Fluxo recomendado

  1. Mensalidade da loja -> Mercado Pago Assinaturas.
  2. Pagamento pontual -> Checkout Transparente Mercado Pago com Pix, cartão e boleto.
  3. Licença da loja -> atualizada pelo servidor após confirmação do pagamento/webhook.
  4. Se a cobrança passar para atraso e a licença vencer, o acesso da loja é bloqueado automaticamente.

4. WhatsApp automático de cobrança

  1. Crie um app na Meta e habilite o WhatsApp Business Platform / Cloud API.
  2. Configure no servidor: WHATSAPP_TOKEN, WHATSAPP_PHONE_NUMBER_ID, WHATSAPP_TEMPLATE_NAME e WHATSAPP_TEMPLATE_LANG.
  3. No WhatsApp Manager, crie e aprove um template com 4 parâmetros no corpo: nome da loja, valor, vencimento e link da loja.
  4. Depois disso, o painel financeiro pode enviar cobrança manual e também executar a rotina automática.
